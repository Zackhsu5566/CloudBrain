# User Profile

> This file is the agent's stable long-term view of the user.
> It starts mostly empty. The agent fills it as it learns things that are
> **persistent, reusable facts** — the kind of info it'd want instantly
> available at the start of every session.
>
> Transient things (today's mood, recent topics, what we talked about
> yesterday) belong in LanceDB via `memory_store`, not here.
>
> When updating this file, the agent should:
> - Keep it terse. Bullet points, not paragraphs.
> - Only add things it's confident are stable (mentioned across sessions,
>   or explicitly framed as "remember this").
> - Revise when facts change. Don't just append.
> - Write in English (per SOUL.md archive rule).
